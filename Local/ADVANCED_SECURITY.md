# 🛡️ Продвинутая Система Безопасности МатЦентра

## 🎯 Обзор

МатЦентр теперь оснащён **профессиональной многоуровневой системой защиты**, включающей:

1. ✅ **SHA-256 хеширование паролей**
2. ✅ **Browser Fingerprinting** (отпечатки устройств)
3. ✅ **Session Management** (управление сессиями)
4. ✅ **Exponential Backoff** (увеличивающееся время блокировки)
5. ✅ **Activity Monitoring** (мониторинг активности)
6. ✅ **Encrypted Storage** (шифрование данных)
7. ✅ **Automatic Session Expiry** (автоматическое истечение сессий)

---

## 🔐 Уровень 1: Хеширование Паролей

### Что это?
Пароль **никогда не хранится в открытом виде**. Вместо этого используется односторонний SHA-256 хеш.

### Как работает?
```javascript
// Пароль: "myPassword123"
// Хеш: "ef92b778bafe771e89245b89ecbc08a44a4e166c06659911881f383d4473e94f"
```

**Преимущества:**
- Даже если кто-то получит доступ к localStorage, пароль невозможно восстановить
- Каждый пароль создаёт уникальный хеш
- Используется стандартный Web Crypto API

## 🔍 Уровень 2: Browser Fingerprinting

### Что это?
Уникальный "отпечаток" браузера/устройства, основанный на:
- User Agent
- Разрешение экрана
- Глубина цвета
- Часовой пояс
- Язык браузера
- Платформа
- Количество ядер процессора
- Объём памяти

### Зачем?
**Обнаружение кражи токена/сессии:**
- Если кто-то скопирует данные из localStorage и попытается использовать их на другом устройстве
- Система обнаружит несоответствие отпечатков
- Сессия будет автоматически аннулирована

```
⚠️ Пример атаки:
Злоумышленник копирует данные из localStorage → 
Пытается войти со своего компьютера → 
Отпечаток не совпадает → 
🚨 Доступ запрещён!
```

## 💳 Уровень 3: Session Management

### Что это?
Вместо постоянного хранения пароля система использует **защищённые сессии**.

### Характеристики:
- **Срок жизни:** бессрочно (не истекает)
- **Автоматическое продление:** не требуется
- **Проверка при каждом запуске:** да
- **Защита от кражи:** проверка fingerprint
- **Удаление:** только при явном выходе

### Структура сессии:
```javascript
{
  token: "случайный 64-символьный токен",
  passwordHash: "SHA-256 хеш пароля",
  fingerprint: "отпечаток устройства",
  createdAt: 1234567890000,
  expiresAt: 1234654290000
}
```

### Что происходит:
1. **При успешном входе:** создаётся новая сессия
2. **При открытии страницы:** проверяется действительность сессии
3. **Если сессия валидна:** автоматический вход
4. **Если истекла/подделана:** запрос пароля

## ⏱️ Уровень 4: Exponential Backoff

### Что это?
**Время блокировки увеличивается** с каждой новой серией неудачных попыток.

### Схема блокировок:

| Блокировка | Время блокировки | После сброса |
|-----------|------------------|--------------|
| 1-я       | **5 минут**      | Через 5 мин  |
| 2-я       | **15 минут**     | Через 15 мин |
| 3-я       | **1 час**        | Через 1 час  |
| 4-я+      | **24 часа**      | Через 1 день |

### Пример сценария:

```
День 1, 10:00 → 3 неудачные попытки → Блокировка на 5 минут
День 1, 10:05 → Разблокировка
День 1, 10:10 → 3 неудачные попытки → Блокировка на 15 минут
День 1, 10:25 → Разблокировка
День 1, 14:00 → 3 неудачные попытки → Блокировка на 1 час
День 1, 15:00 → Разблокировка
День 2, 09:00 → 3 неудачные попытки → Блокировка на 24 часа
День 3, 09:00 → Разблокировка
```

### Сброс счётчика:
Счётчик блокировок сбрасывается **через неделю** после успешного входа.

## 🚨 Уровень 5: Activity Monitoring

### Что отслеживается?
Все попытки входа (успешные и неудачные) записываются с:
- Timestamp (время)
- Успех/неудача
- Fingerprint устройства

### Обнаружение подозрительной активности:

**Критерии подозрительности:**
1. ⚠️ Более **3 разных устройств** пытаются войти
2. ⚠️ Более **7 неудачных попыток** за короткий период

**Что происходит:**
```javascript
🚨 ПОДОЗРИТЕЛЬНАЯ АКТИВНОСТЬ ОБНАРУЖЕНА!
   - Уникальных устройств: 5
   - Неудачных попыток: 9
```

Пользователь видит специальное предупреждение.

### История попыток:
Хранятся **последние 50 попыток** (зашифровано).

## 🔒 Уровень 6: Encrypted Storage

### Что шифруется?
- История попыток входа
- Данные сессии

### Метод шифрования:
**XOR шифрование** с ключом = fingerprint устройства

### Зачем?
- Данные в localStorage не читаются в plain text
- Без fingerprint невозможно расшифровать
- Базовая обфускация от любопытных глаз

**Пример:**
```javascript
// Открытый текст:
{"timestamp": 1234567890, "success": true}

// Зашифрованный:
QmFzZTY0IGVuY29kZWQgWE9SIHN0cmluZw==
```

## ⏰ Уровень 7: Persistent Sessions

### Бессрочные сессии:
- **Срок жизни сессии:** бессрочно (не истекает)
- **Проверка при запуске:** да (проверка fingerprint)
- **Удаление:** только при явном выходе

### Преимущества:
- ✅ Не нужно повторно входить после перезапуска
- ✅ Удобство использования
- ✅ Защита через fingerprint (нельзя украсть сессию)
- ✅ Явный контроль (кнопка "Выйти")

### Что происходит:
```
Пользователь входит →
Создаётся бессрочная сессия →
При каждом запуске проверяется fingerprint →
Если всё ОК → автоматический вход →
Работает бессрочно до нажатия "Выйти"
```

## 🎭 Сценарии Атак и Защита

### 1. Brute-Force атака
**Атака:** Робот пробует тысячи паролей
**Защита:** 
- После 3 попыток → блокировка на 5 минут
- Повторные попытки → exponential backoff
- Детектор подозрительной активности

### 2. Кража localStorage
**Атака:** Злоумышленник копирует данные из localStorage
**Защита:**
- Fingerprint не совпадёт на другом устройстве
- Сессия будет аннулирована
- Хешированный пароль нельзя использовать напрямую

### 3. Session Hijacking
**Атака:** Перехват и использование токена сессии
**Защита:**
- Сессия привязана к fingerprint
- При несоответствии → автоматическая очистка
- Шифрование данных сессии

### 4. Rainbow Table атака
**Атака:** Использование готовых хеш-таблиц
**Защита:**
- SHA-256 с fingerprint-специфичной солью
- Каждое устройство имеет уникальную соль

### 5. Multiple Device атака
**Атака:** Использование множества устройств
**Защита:**
- Детектор подозрительной активности
- Специальное предупреждение
- История со всеми fingerprints

## 📞 Поддержка

### Проблемы?

1. **Не могу войти после обновления:**
   ```javascript
   resetSecurityData()
   ```

2. **Бесконечная блокировка:**
   ```javascript
   localStorage.removeItem('matcenter_lockout_until');
   localStorage.removeItem('matcenter_failed_attempts');
   location.reload();
   ```

3. **Сессия постоянно истекает:**
   - Проверьте системное время
   - Убедитесь, что часовой пояс правильный

4. **"Несоответствие отпечатка":**
   - Нормально при смене браузера/устройства
   - Просто введите пароль заново
